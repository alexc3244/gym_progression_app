# Progression Logic (Hybrid Model)

## Overview

Progression is based on a combination of:
- Rep range completion (deterministic rule), to be filled in by user
- Perceived exertion (RPE/RIR autoregulation)

---

## Deterministic Rules

### Rule 1: Load Increase
If all sets hit top of rep range:
→ increase load by 2.5–5%
- note that this may change with future features (e.g. compounds/ bigger muscle groups change differently)

---

### Rule 2: Load Maintenance
If sets fall within target range but not at top:
→ maintain load

---

### Rule 3: Load Reduction
If failure occurs or reps fall below minimum threshold:
→ reduce load by 2.5–5%

---

## Autoregulation Layer

### RPE Adjustment

- RPE ≤ 7:
  → increase progression rate (faster load increases)

- RPE 8–9:
  → standard progression

- RPE ≥ 9:
  → slow progression or maintain load

---

## Final Weight Calculation

Final load adjustment is:

Base progression (rule-based)
× autoregulation modifier

---

## Philosophy

The system prioritises:
- steady progressive overload
- avoiding excessive fatigue accumulation (a key one as over training is easily done)
- interpretability over complexity
